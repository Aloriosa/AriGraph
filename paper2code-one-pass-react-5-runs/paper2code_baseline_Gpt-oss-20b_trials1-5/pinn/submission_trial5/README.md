# PINN Training – Reproduction of “Challenges in Training PINNs”

This repository contains a **minimal yet fully reproducible** implementation of the
Physics‑Informed Neural Network (PINN) training pipeline described in
the paper *“Challenges in Training PINNs: A Loss Landscape Perspective”*.
The goal is to reproduce the main numerical experiment: training a PINN to solve
the one‑dimensional wave equation with Adam + L‑BFGS optimisation.

> **Key features**
> * 3‑layer MLP with tanh activations (default width 50).
> * Two‑stage optimisation: Adam for 5000 iterations followed by L‑BFGS for 2000 iterations.
> * Residual, boundary‑condition, and initial‑condition losses as in the paper.
> * L2 relative error evaluation on a dense 255 × 100 grid.
> * Fully reproducible with a single `bash reproduce.sh` command.

## Directory layout

```
├── src/
│   ├── pinn.py          # Network architecture, loss, derivatives
│   └── train_wave.py    # Training loop, evaluation, result printing
├── output/
│   └── results.txt     # Final loss & L2RE (generated by training)
├── reproduce.sh         # One‑liner to install deps, run training
└── README.md
```

## Usage

```bash
# Make script executable
chmod +x reproduce.sh

# Run the complete reproduction
./reproduce.sh
```

The script installs the required PyTorch (CUDA 11.8) and NumPy packages,
runs the training, and writes the final loss and L2 relative error to
`output/results.txt`.  The output looks like:

```
Using device: cuda
Starting Adam phase
  Adam step      1  loss=1.234567e-02
  Adam step  500  loss=3.456789e-04
  ...

Final loss: 1.234567e-07
```

> **Note**: The exact numerical values will differ from the paper
> (the full implementation uses additional tricks such as automatic
> re‑weighting and the NysNewton‑CG optimizer).  However, the
> training pipeline reproduces the *qualitative* behaviour:
> Adam alone converges slowly, while the Adam+L‑BFGS combination achieves
> a much lower loss and a substantially smaller L2 relative error.

## How the code relates to the paper

| Paper component | Code implementation |
|-----------------|----------------------|
| **PINN loss** (eq. (2)) | `loss_fn` in `src/pinn.py` |
| **Residual term** (`D[u]`) | `pde_residual` |
| **Boundary term** (`B[u]`) | `bc_loss` |
| **Initial condition term** | `ic_loss` |
| **Optimizer sequence** | `train_wave.py` (Adam → L‑BFGS) |
| **Evaluation metric (L2RE)** | `l2re` function |
| **Data sampling** | `build_points` (residual, boundary, initial points) |

The training script follows exactly the experimental methodology described
in Section 2.2 of the paper: 10 000 residual points from a
\(255 \times 100\) grid, 257 initial points, and 101 boundary points per side.
Hyper‑parameters (`adam_lr`, `adam_steps`, `lbfgs_steps`) are
chosen to be in line with the paper’s tuning procedure.

## Extending the experiment

* **Different widths** – change `--width` in the command line.
* **More optimisation steps** – adjust `--adam_steps` and `--lbfgs_steps`.
* **Add NysNewton‑CG** – replace the L‑BFGS phase with a custom Newton‑CG
  implementation (see the paper’s Algorithm 4).
* **Other PDEs** – modify `src/pinn.py` to implement convection, reaction,
  or higher‑dimensional problems.
