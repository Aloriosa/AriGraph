# BBoxвЂ‘Adapter вЂ“ Lightweight Adapter for BlackвЂ‘Box LLMs

This repository contains a *minimal, fully reproducible* implementation of the idea behind the
**BBoxвЂ‘Adapter** paper.  
The goal is to demonstrate how a small, trainable adapter can be used to
adapt a blackвЂ‘box language model (here, a local `gpt2` model) to a downstream
task **without accessing the internal parameters or token probabilities** of the
base LLM.

> **Note**  
> The experiments here are intentionally lightweight (a few dozen training
> examples).  They are meant to showcase the pipeline, not to reproduce the
> largeвЂ‘scale results reported in the paper.

## How to reproduce

1. **Create a clean environment** (the script will install the dependencies)
2. **Run the reproduction script**  
   ```bash
   bash reproduce.sh
   ```
   This will:
   * Install the required Python packages
   * Train the adapter on the toy dataset (`data/train.csv`)
   * Run inference on `data/test.csv`
   * Write the predicted answers to `output/predictions.csv`

3. **Verify the results**  
   The `predictions.csv` file contains the question, the topвЂ‘1 predicted answer,
   and the adapter score.  The script prints a short summary to the console.

The whole process finishes within a few minutes on a CPUвЂ‘only machine
(adding the `--use_cuda` flag will use the GPU if available).

## Code structure

* **`blackbox_llm.py`** вЂ“ Wrapper around a HuggingFace `GPT2LMHeadModel`
  that only exposes a `generate` method.  It behaves like a blackвЂ‘box
  LLM: we can ask it to produce text, but we never inspect gradients
  or token probabilities.

* **`adapter.py`** вЂ“ A lightweight adapter that scores candidate
  completions.  It is a `DistilBertModel` encoder followed by a linear
  layer that outputs a scalar score.  During training we optimise
  a rankingвЂ‘based NCE loss that encourages the groundвЂ‘truth answer
  to receive a higher score than the LLMвЂ‘generated candidates.

* **`train.py`** вЂ“ Loads the training data, runs the blackвЂ‘box LLM to
  generate negative samples, trains the adapter, and saves the best
  checkpoint.

* **`inference.py`** вЂ“ Loads the checkpoint, generates candidates for
  the test set, scores them with the adapter, and writes the best
  predictions to CSV.

* **`utils.py`** вЂ“ Miscellaneous helper functions (seeding, data
  loading, etc.).
