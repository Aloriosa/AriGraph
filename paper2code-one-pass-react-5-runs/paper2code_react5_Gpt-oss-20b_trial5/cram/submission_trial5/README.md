# Memory Vector Compression Demo  
*Reproduction of “Cramming 1568 Tokens into a Single Vector and Back Again”*

This repository reproduces the core experiments from the paper **Cramming 1568 Tokens into a Single Vector and Back Again: Exploring the Limits of Embedding Space Capacity**.  
The implementation follows the paper’s per‑sample optimisation of trainable memory vectors (`[mem]`) for a frozen language model and evaluates the following metrics:

| Metric | Description |
|--------|-------------|
| **Teacher‑forcing accuracy** | Accuracy of token predictions when the model is fed the ground‑truth prefix. |
| **Free‑generation accuracy** | Accuracy of the sequence generated by the model conditioned only on `[mem]`. |
| **Token Gain** | Difference in the number of correctly predicted tokens with and without `[mem]`. |
| **Information Gain** | Reduction in cross‑entropy (bits per token) when using `[mem]`. |
| **Decoding Capacity** | Largest token length for which the teacher‑forcing accuracy with `[mem]` is **≥ 0.99**. |

The script also evaluates the method on **random word sequences** (to probe the effect of language model prior knowledge) in addition to natural text.

> **Note** – The original paper used large LLMs such as Llama‑3.2‑1B.  
> Running those models requires > 30 GB of GPU memory.  
> The reproduction here uses **GPT‑2‑medium** (345 M parameters) which can be run on a single NVIDIA A10 GPU.  The relative ranking of the metrics is preserved, but the absolute compression ratios are lower.

## Reproduction

```bash
# In the container
bash reproduce.sh
```

The script will:

1. Install system and Python dependencies.  
2. Run `compress.py` for a set of token lengths and numbers of memory vectors.  
3. Generate CSV result files in `results/`.  
4. Produce a small `decoding_capacity.csv` file summarising the decoding capacity for each `k`.

### Customisation

* `--model` – HuggingFace model name (default `gpt2-medium`).  
* `--text-file` – Path to a text file with one sentence per line (used for natural language experiments).  
* `--random` – Generate random word sequences instead of using the text file.  
* `--lengths` – List of token lengths to evaluate.  
* `--k` – List of numbers of memory vectors to try.  
* `--threshold` – Accuracy threshold for decoding capacity (default 0.99).  
* `--epochs` – Number of optimisation steps for each memory vector set.

### Expected Output

CSV files such as `results_len64_k1.csv` contain the following columns:

| Column | Meaning |
|--------|---------|
| `text_length` | Target token length |
| `k` | Number of memory vectors |
| `accuracy_tf` | Teacher‑forcing accuracy |
| `accuracy_free` | Free‑generation accuracy |
| `token_gain` | Tokens gained by using `[mem]` |
| `info_gain_bits_per_token` | Bits per token saved |
| `correct_tokens_mem` | Correct tokens with `[mem]` |
| `correct_tokens_base` | Correct tokens without `[mem]` |
| `loss_bits_base` | Cross‑entropy (bits/token) without `[mem]` |
| `loss_bits_mem` | Cross‑entropy (bits/token) with `[mem]` |

The `decoding_capacity.csv` file lists the maximum length for each `k` that satisfies the accuracy threshold.
