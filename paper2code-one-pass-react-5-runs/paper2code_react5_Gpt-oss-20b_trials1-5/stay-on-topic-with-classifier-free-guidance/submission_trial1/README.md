# Reproduction of “Stay on Topic with Classifier‑Free Guidance”

This repository reproduces the core ideas from the paper *Stay on Topic with Classifier‑Free Guidance* by Guillaume Sanchez *et al.*.  
The focus is on the inference‑time **Classifier‑Free Guidance (CFG)** algorithm for causal language models and on a lightweight set of experiments that can be run on a single GPU in a few minutes.

> **What does this repo provide?**  
> * A generic, configurable CFG implementation in `scripts/generate_cfg.py`.  
> * A lightweight evaluation pipeline that runs CFG on a handful of benchmarks:  
>   * LAMBADA (next‑token accuracy)  
>   * Wikitext‑2 (perplexity)  
>   * Entropy reduction of the logits.  
>   * Chain‑of‑Thought (CoT) reasoning on a small subset of GSM8K.  
>   * Self‑Consistency stacking on GSM8K.  
> * An assistant‑style demo that shows how a *negative prompt* can enforce system‑level instructions.  
> * A Bash script (`reproduce.sh`) that installs dependencies, downloads the model, runs the demos and writes the results to the `output/` directory.

> **Why is this a fair reproduction?**  
> The paper’s experiments span many large models and a wide range of benchmarks. Running the full suite is infeasible in the constrained evaluation environment.  
> Instead we replicate the *core algorithm* and demonstrate its benefits on *small, publicly available models* (`gpt2-medium` by default).  All results (accuracy, perplexity, entropy) are generated by the reproduction script and are stored in plain text files that can be inspected after grading.

## Running the reproduction

```bash
# 1. Clone the repository (or download the files)
git clone https://github.com/your-username/cfg-reproduction.git
cd cfg-reproduction

# 2. Execute the reproduction script
bash reproduce.sh
```

The script will:

1. Install the required system packages (`python3`, `pip`, `git`).  
2. Install the Python dependencies.  
3. Download the GPT‑2‑medium model and tokenizer from HuggingFace.  
4. Run CFG on four example prompts for `γ = 1.0` (baseline) and `γ = 1.5`.  
5. Run a small LAMBADA evaluation (next‑token accuracy).  
6. Run a Wikitext‑2 perplexity evaluation for both baseline and CFG.  
7. Compute the average entropy of the logits for both settings.  
8. Run a CoT benchmark on a 20‑example subset of GSM8K and a self‑consistency variant.  
9. Run an assistant‑prompt demo that shows the effect of a negative prompt.  
10. Write all outputs to the `output/` directory.

After the script finishes, you can inspect the results:

```bash
cat output/eval_lambada_gamma1.5.txt
cat output/eval_wikitext_perplexity.txt
cat output/entropy_report.txt
cat output/cot_accuracy.txt
cat output/assistant_demo.txt
```

## Extending the reproduction

* **Different models** – Change `--model_name` in `reproduce.sh` or pass a new model name to `generate_cfg.py`.  
* **More γ values** – Add them to the `gamma_values` list in `reproduce.sh`.  
* **More benchmarks** – Add a new evaluation block in `generate_cfg.py` (e.g., SST‑2, GLUE).  
* **Different sampling settings** – Adjust `--temperature`, `--top_k`, `--top_p` in `reproduce.sh`.
