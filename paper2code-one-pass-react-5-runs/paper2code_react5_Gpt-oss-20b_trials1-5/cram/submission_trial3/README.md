# Reproduction of “Cramming 1568 Tokens into a Single Vector and Back Again”

This repository implements the core idea of the paper *Cramming 1568 Tokens into a Single Vector and Back Again* by performing a per‑sample optimisation of trainable memory vectors (`[mem]`) for a frozen large language model.

## What we reproduce

* **Per‑sample optimisation** of a small set of trainable vectors that encode an entire text sequence.
* **Teacher‑forcing evaluation** to compute token‑level accuracy, token‑gain and information‑gain.
* **Decoding capacity**: the longest length for which the teacher‑forcing accuracy of the `[mem]` run exceeds a user‑defined threshold.
* **Theoretical capacity bound** from Eq. (1) in the paper.
* **Linear scaling** of capacity with the number of trainable vectors.
* Experiments on **natural text** (a long repeated sentence) and on **random token sequences**.
* Persisting the best `[mem]` vectors for each experiment.

## Repository layout

| File | Purpose |
|------|---------|
| `compress.py` | Main executable that runs all experiments and writes a comprehensive `results.json`. |
| `sample_text.txt` | A longer (≈ 2000 tokens) repeating sentence used for the default natural‑text experiment. |
| `config.json` | JSON file describing the experiments that `compress.py` will run. |
| `requirements.txt` | Python dependencies (PyTorch, transformers, tqdm). |
| `reproduce.sh` | Bash script that installs the environment and runs `compress.py`. |
| `results.json` | Output file containing detailed results for every experiment. |

## Running the reproduction

```bash
bash reproduce.sh
```

The script will:

1. Install the required packages (PyTorch with CUDA 12.1, transformers, etc.).  
2. Run `compress.py` with the experiments defined in `config.json`.  
3. Write a detailed `results.json` file and save the best `[mem]` vectors for each experiment.

The output file contains, for each experiment, the following keys:

```json
{
  "model": "...",
  "mem_vectors": 1,
  "length": 512,
  "theoretical_capacity_tokens": 1024,
  "decoding_capacity_tokens": 512,
  "best_accuracy_with_mem": 1.0,
  "baseline_accuracy": 0.1,
  "token_gain": 512.0,
  "information_gain": 3.2,
  "cross_entropy_with_mem": 1.1,
  "baseline_cross_entropy": 5.3,
  "compression_ratio": 512.0,
  "actual_vs_theoretical_ratio": 0.5,
  "mem_vector_path": "mem_Llama-3.2-1B_512_1.pt"
}
```

Feel free to edit `config.json` to add more models, change the number of `[mem]` vectors, or experiment with different text lengths and domains.

---

**Important notes**

* The script uses **teacher‑forcing** for both the `[mem]` and baseline runs, matching the methodology described in the paper.
* The **decoding capacity** metric is computed as the longest length for which the teacher‑forcing accuracy of the `[mem]` run is above the `threshold` defined in `config.json` (default 0.99).
* Random token sequences are generated by sampling from the model’s vocabulary (excluding special tokens) and are used to evaluate the language‑agnostic capacity of the `[mem]` vectors.
* The script requires a GPU with CUDA 12.1; it will fallback to CPU if a GPU is not available.
* Best `[mem]` vectors are saved to a file named `mem_[model]_[length]_[mem_vectors].pt`.
* The ratio of the achieved length to the theoretical capacity (`actual_vs_theoretical_ratio`) is reported for each experiment.
