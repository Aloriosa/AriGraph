# Reproduction of Sample-Specific Masking (SMM) for Visual Reprogramming

## Overview
This repository reproduces the Sample-Specific Masking (SMM) method from the paper "Sample-Specific Masking for Visual Reprogramming" (Tsao et al., 2024). SMM is a parameter-efficient method for adapting pre-trained vision models to new tasks by learning sample-specific masks that modify inputs before they enter the frozen pre-trained model.

The key innovation of SMM is replacing the shared mask used in traditional visual reprogramming with a sample-specific mask generated by a lightweight convolutional network. This allows the mask to adapt to individual input images, reducing approximation error and improving performance.

## Implementation Details
- **Method**: Sample-Specific Masking (SMM) as described in the paper
- **Base Model**: ResNet-18 (pre-trained on ImageNet)
- **Dataset**: Oxford Pets (37 classes)
- **Input Size**: 224×224
- **Mask Generator**: Lightweight CNN with 5 layers, 3 output channels
- **Patch Size**: 8×8
- **Training**: Adam optimizer, learning rate 0.001, batch size 32, 100 epochs
- **Loss**: Cross-entropy

## Key Components
1. **AttributeNet**: Lightweight CNN that generates sample-specific masks from input images
2. **InstancewiseVisualPrompt**: Main module that combines the mask with a learnable pattern and applies it to input images
3. **Patch-wise Interpolation**: Upsamples the low-resolution mask to match input image dimensions
4. **Label Mapping**: Maps source task labels to target task labels for the pre-trained model's output

## Reproduction Results
Running `reproduce.sh` will:
1. Download and preprocess the Oxford Pets dataset
2. Train the SMM model for 100 epochs
3. Evaluate the final model on the test set
4. Generate evaluation results

Expected results (based on paper):
- Accuracy on Oxford Pets: ~85-87% (improvement over shared mask methods)
- Training time: ~2-3 hours on NVIDIA A10 GPU
- Model size: ~0.54M parameters (only the mask generator is trainable)

The results will be saved in `/home/submission/results/` with:
- `best_model.pth`: Trained model checkpoint
- `evaluation_results.txt`: Detailed evaluation metrics
- `final_results.txt`: Summary of achieved performance
