# Stochastic Interpolants with Data-Dependent Couplings Reproduction

This repository implements the Stochastic Interpolants framework with data-dependent couplings as described in the paper. The implementation focuses on image inpainting and super-resolution tasks using continuous-time generative modeling.

## Implementation Overview

The implementation follows the paper's core components:

1. **Stochastic Interpolants**: A continuous-time generative pipeline that interpolates between a base density (standard Gaussian) and target density (image data) using a time-dependent stochastic process.

2. **Data-Dependent Couplings**: Instead of using a fixed base density, we construct a data-dependent coupling where the base samples are generated by corrupting target samples with Gaussian noise according to the formula: `x₀ = x₁ ⊕ (1 - mask) ⊙ ε`, where:
   - `x₁` is a target image sample
   - `mask` is a binary mask defining the region to preserve (inpainting) or downsample (super-resolution)
   - `ε` is Gaussian noise
   - `⊕` denotes element-wise operation

3. **Velocity Field Estimation**: We learn a velocity field via square loss regression on paired samples from the base and target densities, using a U-Net architecture to predict the velocity at each time step.

4. **Training**: The model is trained using the square loss objective: `L = E[||v_θ(x_t, t) - v(x_t, t)||²]` where `v(x_t, t)` is the true velocity field derived from the data-dependent coupling.

## Reproduction Process

1. **Training**: Run `reproduce.sh` to train models for both inpainting and super-resolution tasks
2. **Sampling**: The script automatically generates samples using the learned velocity fields
3. **Evaluation**: FID scores and PSNR metrics are computed for quantitative assessment

## Expected Results

After running `reproduce.sh`, you should find:

1. **Inpainting results**:
   - Trained model checkpoint at `results/inpainting/checkpoint_best.pth`
   - Generated samples at `results/inpainting/samples/`
   - FID score reported in `results/inpainting/metrics.txt`

2. **Super-resolution results**:
   - Trained model checkpoint at `results/super_resolution/checkpoint_best.pth`
   - Generated samples at `results/super_resolution/samples/`
   - PSNR and FID scores reported in `results/super_resolution/metrics.txt`

The implementation should achieve results comparable to the paper's reported performance, with the data-dependent couplings outperforming independent coupling baselines in both tasks.

## Implementation Notes

- We use the CelebA dataset for inpainting and DIV2K for super-resolution
- The noise schedule is linear as specified in memory_card_0093
- The U-Net architecture follows the standard design with attention mechanisms as suggested by memory_card_0052
- Training uses gradient descent with Adam optimizer as per paper_card_0006
- The base density is configured as a standard Gaussian distribution as specified in paper_card_0002
- The target density is accessed via empirical samples only as per memory_card_0004
- We use the boundary conditions t=0 for base and t=1 for target as per memory_card_0172
